# Backlog GoDo-App

## Контекст

GoDo — мобильный сервис поиска и записи на мастер-классы. Этот документ описывает карточки первой версии продукта (MVP) — то, что нужно сделать, чтобы пользователь смог пройти сквозной сценарий: открыть приложение, найти подходящий мастер-класс, открыть карточку и оставить заявку.

Документ — source of truth для backlog. Карточки в YouGile дублируют его заголовки, ссылка на этот файл указана в карточках. Изменения в backlog (новые задачи, переформулировки, отмена) сначала прилетают сюда через PR, потом синхронизируются в YouGile.

### Структура карточки

- **Цель** — зачем задача нужна продукту/команде, какую проблему закрываем.
- **Что сделать** — конкретные шаги исполнителя.
- **Критерии готовности (DoD)** — условия, при которых задачу можно закрывать.
- **Зависимости** — что должно быть готово до неё и что она блокирует.
- **Артефакты** — что появится по итогу (PR, документ, прототип).

### MVP-сценарий

```
Запуск приложения
    → Каталог мастер-классов
        → Поиск/фильтры
            → Карточка мастер-класса
                → Форма записи
                    → Подтверждение записи
```

Чего сознательно НЕ делаем на MVP: онлайн-оплата, личный кабинет, отзывы, чат с организатором, push-уведомления, рекомендации.
